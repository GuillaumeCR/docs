About our Products

The Polymer Project team builds a variety of libraries and tools for developers. The following major products are under active development::

  • LitElement . An ultra-light, highly performant custom element base class with a simple but expressive API.
  • PWA Starter Kit . A set of starting templates for building full-featured Progressive Web Apps using web components and optionally Redux for state management.
  • Material Web Components . High-fidelity versions of Google's Material Design spec that work anywhere on the web.
  • Polymer CLI and related tools . Development tools that help with scaffolding out projects, serving, testing, and building projects for production.
  • lit-html library . A small, lightweight templating library, used by LitElement.

The Polymer Project also maintains the following products:

  • Polymer Classic library 3.x . Our original web components library (also known as the Polymer library or simply "Polymer"). Version 3.x is updated to use JavaScript modules and npm, for seamless interop with popular frameworks and tools.
  • Polymer Classic library 1.x and 2.x . Previous versions of the library. Versions 1.x and 2.x use HTML imports (a proposed web standard which is no longer on track to be standardized), and are distributed using Bower. Version 1.x also uses an older set of web components standards.
  • Polymer element collections (paper-, iron- and gold- elements). Two major versions of the elements are currently supported: a version compatible with Polymer classic library 3.x, and a hybrid version that works with version 1.x and 2.x of the library.

These projects are in maintenance mode: we're fixing bugs, but we're not doing new feature work on them. For more information on our product lifecycle, see How we support our products.

But what should I use?

That's a lot of products to think about! How do you know what to use? He's a quick summary of what we recommend for some common use cases.

If you are... We recommend...
Developing a new element from scratch. LitElement. It provides a simple, performant, and lightweight base class.
Developing a new app from scratch. PWA Starter Kit. It provides a number of templates for starting a modern Progressive Web App using web components.
Updating an app written using Polymer 1.x or 2.x elements. Polymer 3.x. Moving to Polymer 3 removes the need for the HTML imports polyfill. It also makes it easier to work with existing ES modules distributed using npm; and to use third-party tools.

An app built using Polymer 3 elements can more easily take advantage of new elements distributed as ES modules, including the Material Web Components.
Maintaining an app written using Polymer 1.x or 2.x elements, with no feature work planned. Polymer 2.x. If you don't plan any new feature work on your app, you probably don't need to migrate to Polymer 3.x.

If you're still on Polymer 1.x, we recommend upgrading to 2.x to take advantage of the native web components v1 support that's shipping in Chrome and Safari (and in progress on other browsers). Polymer 1.x relies on older web components standards that are only implemented in Chrome.
Creating your own library or base class. lit-html. It provides a small, lightweight, and efficient templating library for the web.
Looking for material design UI elements to use with a Polymer 3.x or LitElement based app. Material Web Components. These elements are more performant than the earlier paper elements, and follow the latest version of the web components spec.
Looking for a material design UI element that isn't provided by MWC yet. Paper elements or third-party component. Almost all of the paper elements are available in 3.x-compatible versions.
